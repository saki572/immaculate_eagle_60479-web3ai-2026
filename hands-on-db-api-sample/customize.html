🎨 Customize

カスタマイズして遊ぶ

Step 1〜3 で動いたアプリを 自分だけのアプリに変えていきましょう。
minta先生 の人格を変える、別の API と組み合わせる、DB スキーマを拡張する など、レシピ集。

🌱 Step 3 完了が前提 🎯 自分のオリジナル作品を作る
「動くサンプル」から「自分の作品」へ

ハンズオンで作った「教えて、minta先生」は、あなたが書き換える前提のテンプレです。
Claude Code と対話しながら、思いついた改造をどんどん試してください。動かなくなっても git revert で戻せます。

🌱 やさしいカスタマイズ（10〜30分）

🎭 minta先生を別キャラに変える

編集箇所: src/lib/minta.ts の MINTA_SYSTEM_PROMPT

例: 「冷静沈着な執事 Sebastian」「江戸時代の蘭学者」「未来から来たロボット教師」「猫」「あなたの推しキャラ」など。回答のトーンや一人称、口調を書き換えるだけで全く別物になります。

📋 Claude Code に投げるプロンプト例 
src/lib/minta.ts の MINTA_SYSTEM_PROMPT を、以下のキャラに合わせて書き換えてください。

# キャラ設定
- 名前: __ (例: Sebastian)
- 役柄: __ (例: ヴィクトリア朝の執事)
- 口調: __ (例: 丁寧・古風・第三人称)
- 専門領域: __ (例: 科学とテクノロジー全般)
- NG: 個人情報を扱わない / 政治・宗教には触れない

人格定義を書き換えたら、画面のヘッダー文字列「教えて、minta先生」も、新しいキャラ名に合わせて src/app/page.tsx を更新してください。

🎨 見た目の色を変える

編集箇所: src/app/page.tsx の Tailwind CSS クラス

例: ピンク (rose-*) を ブルー (sky-*) や グリーン (emerald-*) に。Tailwind 4 のカラーパレットは 公式ドキュメントで確認できます。

📋 プロンプト例 
src/app/page.tsx 内の Tailwind カラー (rose-50, rose-500 など) を、すべて __ (例: sky) に置き換えてください。
hover の色も同様に変更してください。

📝 質問のテンプレを変える

編集箇所: src/app/page.tsx の placeholder 文字列

「例: Next.js の App Router と Pages Router の違いを教えてください」を、自分のターゲットユーザーが聞きたくなるような例に書き換える。

🌿 中級カスタマイズ（30〜60分）

🧪 DB スキーマを拡張する

編集箇所: assets/supabase-schema.sql + src/lib/supabase.ts の型 + src/app/api/ask/route.ts

例: sentiment (感情: 喜び/怒り/悲しみ/驚き/その他) のカラムを追加。LLM 分析プロンプトで感情判定を加え、保存する。

📋 プロンプト例 
qa_logs テーブルに sentiment カラム (text型) を追加してください。

具体的にやってほしいこと:
1. assets/supabase-schema.sql に ALTER TABLE 文を追加 (既存テーブルにカラム追加)
2. Supabase の SQL Editor で実行する SQL を出力 (私が手動で実行する)
3. src/lib/supabase.ts の QaLog interface に sentiment を追加
4. src/lib/pii.ts の AnalysisResult interface に sentiment を追加
5. src/lib/pii.ts の ANALYZER_INSTRUCTION で sentiment を判定するよう指示を追加
6. src/app/api/ask/route.ts で sentiment を insert に含める
7. src/app/page.tsx の履歴表示で sentiment をバッジ表示

セキュリティ注意: ALTER TABLE は本番DB で慎重に実行する

📈 質問の集計ダッシュボードを追加

新規ページ: src/app/stats/page.tsx

カテゴリ別の質問数、時間帯別の分布、人気トピックなどを集計・グラフ表示。簡単な統計ダッシュボードが作れます。
chart.js / recharts などのライブラリを Claude Code に install してもらう。

🌍 言語切り替え対応

編集箇所: src/lib/minta.ts の system prompt + UI 文字列

英語版を作って、ヘッダーで「日本語 / English」切り替え。海外の友人にも触ってもらえる教材になる。

🔔 質問数が一定を超えたら通知

新規: Vercel Cron + Slack / Discord Webhook

1時間ごとに DB をチェック → 質問数が増えていれば管理者に通知。
Vercel Cron Jobs (Hobby プランは月100回まで無料) を使えば実装できます。

🎨 別アプリへ転用

この骨格は 「ユーザー入力 → LLM 加工 → DB 蓄積」パターンの最小サンプル。以下のように別ジャンルへ転用できます。

  • 英作文添削 BOT: 質問 → 英作文の評価とリライト → 評価点と分野を DB に保存
  • 料理レシピ提案: 「冷蔵庫にある食材」入力 → レシピ提案 → 人気食材を集計
  • キャリア相談 BOT: 「悩み」入力 → アドバイス → 悩みカテゴリの傾向分析
  • web3 用語解説 BOT: 「分からない単語」入力 → 解説 → よく分からない単語のランキング
  • 授業フィードバック収集: 「今日の授業の感想」 → 要約 → 講師向けにダッシュボード

🛡 カスタマイズ時のセキュリティ要点

  • 新しい API を追加するとき: API キーを必ず環境変数経由で受け取る。コードに直接書かない。
  • DB スキーマ変更: ALTER TABLE は本番に流す前に必ず Supabase の Database Backups を確認。
  • 認証を追加するとき: パスワードリセット導線、Email 認証、CSRF 対策まで含めて設計。
  • 外部からデータを受け取る API: 必ず入力バリデーション。長さ制限・型チェック・SQL injection / XSS 対策。
  • PII を扱う場合: 「保存しない」「ログに出さない」を初期設定として、必要な場合だけ明示的に解除。
  • 本番運用前のセキュリティ レビュー: Claude Code に /security-review スラッシュコマンド (もしあれば) を使う or 「セキュリティ上の懸念点をリストアップして」と頼む習慣を付ける。

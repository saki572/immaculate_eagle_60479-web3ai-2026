🧠 Step 1

ローカル AI を作る

自分の PC の中で LLM (Ollama) と AI エージェント (Hermes) を立ち上げ、CLI で翻訳が動くところまで。LINE 連携は次の Step 2 で。

⏱ 約 20〜30 分 🎯 ゴール: hermes -z "こんにちは" で英訳が返ってくる
これは「目安」です

以下に書いてある推奨セットや手順はあくまで 出発点。受講者の OS・メモリ容量・ネットワーク環境・既存ツールによって、最適解は人それぞれ違います。

Claude Code と対話しながら、自分の PC で動く構成を一緒に探してください。エラーが出たらそのままチャットに貼って「直して」と頼めば良いです。つまずきも学びです。

🪟 Windows をお使いの方へ

Hermes Agent ネイティブ Windows 版は早期ベータで一部機能制限あり。WSL2 (Ubuntu) 上での実行を強く推奨します。WSL2 内では Mac と同一手順がそのまま通ります。

WSL2 未導入の場合、下の起動プロンプトを Claude Code に貼ると、Claude Code がまず OS を判定し、必要なら WSL2 のインストールから一緒に伴走してくれます。PowerShell を開いて wsl --install を実行 → Ubuntu を起動 → そこから改めて Claude Code を開いて続行、という流れになります。

🎯 このステップでやること

  1. 環境を把握する

    OS・CPU アーキ・メモリ・既存ツールを Claude Code に調べてもらい、自分の PC に合うモデルサイズを決める

  2. ローカル LLM ランタイムを入れる

    推奨: Ollama。代替: LM Studio, llama.cpp, vLLM など

  3. 翻訳に使うモデルを取得

    推奨: qwen2.5:3b (多言語・1.9GB)。低スペックなら 1.5b、教材的に崩壊例を見たいなら 0.5b

  4. AI エージェント (Hermes Agent) を入れる

    LLM の上に立つオーケストレータ。LINE プラットフォームへの接続部品を内蔵している

  5. 翻訳人格 (SOUL.md) を設定

    「日↔英の翻訳しかしない」キャラを Hermes に与える

  6. CLI から翻訳できるか確認

    hermes -z "こんにちは" で英訳が返ってくれば Step 1 完了

🟢 推奨スタック (Starter)

動作確認済みの組み合わせ。これで困らなければ、これでいきましょう。

OSmacOS (Apple Silicon) / Linux / WSL2
LLM ランタイムOllama (sudo 不要のインストールが可能)
モデルqwen2.5:3b (約 1.9GB、多言語対応、tool-calling 可)
エージェントHermes Agent v0.14+ (LINE adapter 同梱)

必要ディスク5GB 以上の空き

Step 1 のゴール: 私の PC 上で「ローカル LLM + AI エージェント」を動かし、CLI で翻訳が返ってくる状態にすること。

【最初に必ず: 環境チェックと OS 判定】

何かを実行する前に、必ず以下を私に丁寧に確認してください:
- 私の OS (macOS / Linux / Windows ネイティブ / WSL2 のどれか)
- CPU アーキ・メモリ容量・ディスク空き容量
- 既にインストールされているツール: ollama, hermes, cloudflared, node, brew, python3, curl, unzip

【もし私の OS が Windows ネイティブだった場合】

Hermes ネイティブ Windows 版は早期ベータで動作が不安定なので、まず WSL2 への切り替えを提案してください。具体的には:
1. 「Hermes は WSL2 (Ubuntu) 上で動かす方が今のところ安定です。WSL2 を入れて、その中でこの先を続けるのはどうですか？」と私に確認
2. 私が同意したら、PowerShell (管理者) で wsl --install を実行する手順を案内 (5〜10 分の再起動を含む)
3. Ubuntu が起動したら、その中で Claude Code を開き直してもらい、ステップ 1 を再開する旨を案内
4. もし私が「ネイティブ Windows のまま進めたい」と返したら、その意思を尊重して進めるが、想定される制約 (ダッシュボード機能制限、一部 path 問題) を一度だけ簡潔に伝える

【進め方】

1. PROMPTS.md の P0, P1, P2 を runbook として参照する (機械的に従わず、私の環境に合わせて判断)
2. まず上記の環境調査を実施し、推奨モデルサイズ (0.5b / 1.5b / 3b) を判定して計画を提示
3. 私の OK を得てから順次実行 (各ステップ前に何をするか日本語で簡潔に説明)
4. Ollama 導入 → モデル pull → 単体翻訳テスト
5. Hermes Agent 導入 → Ollama を provider として接続 → SOUL.md に翻訳人格を書く
6. CLI で 4 件のテスト文を翻訳し、すべて成功することを確認
7. エラーは TROUBLESHOOT.md を参照して自分で診断・修復

🆘 よくある詰まり方

"context window 32K is below minimum 64K" エラーが出る

Hermes は最小 64K 要求、qwen2.5 系は native 32K。config.yaml に context_length: 65536 と ollama_num_ctx: 65536 を追加して RoPE 拡張で対応。Claude Code に「context window エラーが出た」と伝えれば自動修復してくれる。

翻訳結果が中国語で返ってくる / 全然違うことを答える

モデルが小さすぎる可能性が高い (0.5b など)。1 ランク上のモデルに切り替えるか、SOUL.md の例示をもっと多くする。Claude Code に「品質が崩壊してる、対処して」で OK。

hermes コマンドが SyntaxError で起動しない

macOS の場合、システム Python 3.9 が拾われてエラーになりがち。正しいパスは ~/.hermes/hermes-agent/venv/bin/hermes。~/tools/bin/hermes にシンボリックリンクを張り直せば解決。

Ollama のダウンロードが遅い・止まる

ネットワーク or ストレージ問題。Claude Code に「ollama pull が止まってる」と伝える。途中まで落ちていれば resume できる。完全に止まる場合は 0.5b のような軽量モデルで代替検討。

全部やり直したい

TROUBLESHOOT.md の P-PRESET-1 プロンプトを Claude Code に貼ると、削除と再構築の手順をガイドしてくれる。

📍 前提

Step 1 を完了して、Bot との DM で minta-bot が応答する状態になっていること。
まだなら先に Step 1 を済ませてください。

Step 2 は AI 全任せフェーズ

Step 1 で手動操作が中心だったのに対し、Step 2 は ほぼ全部 Claude Code がやってくれます。あなたは Discord でメンションして動作確認するだけ。

裏では ncl コマンドで wiring を作ったり、pnpm patch で Discord adapter にパッチを当てたり、CLAUDE.local.md を書き換えたりしています。中身が気になったら Claude Code に「いま何やってるの？」と聞けば全部説明してくれます。

🎯 このステップでやること

  1. 公開チャンネルでの反応を有効化（wiring）

    Bot が DM だけでなく、サーバの公開チャンネルでもメンションに応答するように設定

  2. スレッド自動作成を無効化（pnpm patch）

    メンション時に Discord スレッドが作られてしまう挙動を停止。チャンネルで直接返信させる

  3. 他ユーザーも話せるように開放

    新規ユーザーが話しかけたとき承認カード DM が飛ばないように、unknown_sender_policy=public へ

  4. minta の人格を注入（CLAUDE.local.md）

    テンプレ assets/CLAUDE.local.md.minta を Bot のエージェントグループに上書きコピー

  5. 動作確認（PV）

    DM と公開チャンネルでメンションし、minta が「あざーーす！」など人格通り返すかチェック

P5 / Step 1 / 4

公開チャンネルでも反応するようにする（ワイヤリング）

セットアップ直後の nanoclaw は DM では動くが公開チャンネルでは反応しない状態です。原因は「Owner ロール未登録」「wiring 未作成」「destinations 未登録」のいずれか。./bin/ncl で 4 つの設定（roles / wirings / destinations / messaging-groups policy）を一括で入れます。所要 5〜10 分。

📋 P5 プロンプト
公開チャンネル設定 
ステップ 5: nanoclaw に Discord サーバとエージェントを正しく結びつける。

セットアップ直後の nanoclaw は、Bot との DM では応答するが、公開チャンネルでは反応しないことが多いです。原因は以下のうちどれか:
- Owner ロールがDBに未登録（DMでは Owner として認識されるが、公開チャネルでは別判定）
- messaging_groups と agent_groups の wiring が未作成
- agent_destinations にチャネルが未登録（エージェントが返信先を知らない）

これらを `./bin/ncl` で一括解決します。

1. ~/Documents/nanoclaw に移動

2. 現状を把握:
   ./bin/ncl users list
   ./bin/ncl messaging-groups list
   ./bin/ncl wirings list
   ./bin/ncl destinations list
   ./bin/ncl roles list

3. 私の Discord ユーザー ID を users list から特定。形式は「discord: + 18 桁前後の数字」。確認のため私に Discord ユーザー名を聞き、users テーブルの name 列と照合する。以後のコマンドでは --user に「discord:」接頭辞を含めた完全な ID を渡す。

4. 公開チャンネル用の messaging_group ID（mg-... 形式）を messaging-groups list から特定。channel_type=discord かつ is_group=1 のものが対象。

5. nanoclaw が作った唯一の agent_group ID（minta-bot 用、ag-... 形式）を ./bin/ncl groups list で取得。

6. 「list → 存在しなければ create」の順で、以下を実行:

   # 6-A. Owner ロール（global）
   ./bin/ncl roles grant --user discord: --role owner

   # 6-B. 公開チャンネル wiring（mention モード）
   ./bin/ncl wirings create --messaging-group-id  --agent-group-id  --engage-mode mention

   # 6-C. destination 追加（自動作成されないケースに備えて明示 add）
   #    は半角英数とハイフンで自由に命名（例: my-class-server）
   ./bin/ncl destinations add --agent-group-id  --local-name  --target-type channel --target-id 

   # 6-D. 誰でも話せるように
   ./bin/ncl messaging-groups update --id  --unknown-sender-policy public

7. 終わったら roles list / wirings list / destinations list / messaging-groups list で 4 つすべて反映されていることを確認

8. 私に「Discord の公開チャンネルで @minta-bot こんにちは と打ってください」と促し、応答が来るか確認してもらう

⚠️ unknown_sender_policy=public にすると誰でもメンションで Claude を呼べる = API 料金は所有者持ちです。注意喚起してください。

▶ Claude Code が「公開チャンネルでも応答するようになった。P6 に進んでいい?」と確認してきたら、Discord で @your-bot こんにちは を試して応答することを確認 → 「OK、進めて」と返答 → 下の P6 を開く。

P6 / Step 2 / 4

メンション時のスレッド自動作成を無効化（pnpm patch）

デフォルトでは公開チャンネルでメンションするとスレッドが自動で作られ、その中で返信する仕様です。チャンネルで直接返信させたいので、@chat-adapter/discord パッケージにパッチを当てます。pnpm patch で恒久化されるので、パッケージ更新でも消えません。所要 5〜10 分。

📋 P6 プロンプト
スレッド作成を無効化するパッチ 
ステップ 6: Discord adapter にパッチを当て、メンション時のスレッド自動作成を無効化する。

デフォルトの @chat-adapter/discord は、サーバの公開チャンネルでメンションされると自動的にスレッドを作成し、その中で返信します。これは長い会話の整理には便利ですが、教材や講座では「メンション → そのチャンネルに直接返信」のほうがシンプル。

nanoclaw 流に pnpm patch でパッチを当てます。

1. ~/Documents/nanoclaw に移動

2. パッチ対象の確認:
   node_modules/@chat-adapter/discord/dist/index.js の line ≈801 付近に「if (!discordThreadId && isMentioned)」ブロックがあり、その中で createDiscordThread() を呼んでいる。これを削除すれば、エージェントは元のチャネル直下で返信するようになる

3. pnpm patch を起動:
   pnpm patch @chat-adapter/discord@4.26.0
   → 出力された一時ディレクトリのパスをメモ

4. その一時ディレクトリの dist/index.js を開き、「if (!discordThreadId && isMentioned) { ... createDiscordThread ... }」ブロックを削除。
   削除した位置に「// NANOCLAW-PATCH: do not auto-create a Discord thread on mention. Reply directly in the parent channel instead.」というコメントを残す

5. パッチをコミット:
   pnpm patch-commit '<一時ディレクトリのパス>'
   → patches/@chat-adapter__discord@4.26.0.patch が生成され、pnpm-workspace.yaml の patchedDependencies に登録される

6. nanoclaw ホストを再起動して、パッチ済みの adapter をロードさせる:
   SLUG=$(launchctl list | awk '/com.nanoclaw-v2-/ {gsub(/com.nanoclaw-v2-/,"",$3); print $3}')
   echo "Detected nanoclaw install-slug: $SLUG"
   launchctl kickstart -k "gui/$(id -u)/com.nanoclaw-v2-$SLUG"

7. 再起動完了を待ち、「Discord Gateway connected」がログに出るのを確認

8. 私に「Discord で @minta-bot もう一度 hi と打ってください」と指示し、スレッドを作らずに親チャンネルで直接返答されるか確認

▶ Claude Code が「パッチを当ててホストを再起動した。P7 に進んでいい?」と確認してきたら、Discord で @your-bot こんにちは を試してスレッドが作られず直接返信されることを確認 → 「OK、進めて」と返答 → 下の P7 を開く。

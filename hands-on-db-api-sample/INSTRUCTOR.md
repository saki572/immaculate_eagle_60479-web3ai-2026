## 1. この教材の設計思想

- **「動くものを最速で作る」体験**を最優先。理屈は手を動かした後で吸収する。
- **Claude Code との対話**を前提にする。受講者は1人で詰まらない。
- **無料で完結**できる構成（Gemini Free Tier / Vercel Hobby / Supabase Free / GitHub Free）。
- **セキュリティの基本動作**（環境変数・`.gitignore`・RLS）を初手から組み込む。
- **PII を保存しない設計**を最初に体験させる。AI と DB を組み合わせる場面で必ず出る論点。

## 2. 対象受講者像

### 想定する受講者
- プログラミング経験 **0 〜 半年程度**
- Web の基本（ブラウザ、URL、HTML / CSS の何となく）は知っている
- ターミナル（コマンドライン）を使うのは初めて or 数回程度
- AI と協業して何かを作ってみたい

### 想定しない受講者
- 既に Next.js / Vercel / Supabase を業務で使っている人 → このレベルだと退屈
- フルスタック開発者 → カスタマイズパートのみ参考にする使い方ならOK

### 必要な前提知識・スキル
- 文字を読む集中力（教材は文字ベース）
- 詰まったときに Claude Code に投げる勇気
- 完璧主義を捨てる気持ち（「動けば良い」マインド）

## 3. 所要時間と授業設計の目安

### 90 分授業の場合（推奨）

| 時間 | 内容 |
|---|---|
| 0〜10 分 | スライドで「DB連携・API連携」の概念説明（このページのトップ画像のような図） |
| 10〜30 分 | Step 1（ローカル起動） — 全員で同じ画面まで進める |
| 30〜50 分 | Step 2（Vercel デプロイ） — 公開URL取得 |
| 50〜70 分 | Step 3（Supabase 連携） — DB蓄積を確認 |
| 70〜85 分 | カスタマイズ自由時間 — 好きなレシピを試す |
| 85〜90 分 | 振り返り＋質疑応答 |

### 半日（4時間）授業の場合

- 各 Step に十分な時間を取り、詰まる受講者をフォローしながら進める
- カスタマイズパートを2時間取り、それぞれの作品を全員でシェアする時間に

### 自習用の場合

- 受講者のペースで進めてOK。詰まったら次回授業まで持ち越し
- Discord / Slack で質問チャンネルを用意すると効果的

## 5. 改変のヒント

### キャラ・テーマを自分の文脈に合わせる

- **「minta先生」を自校のマスコットに**変える: `src/lib/minta.ts` の人格定義を書き換え
- **「web3・AI概論」を自分の授業名に**変える: 各 HTML / MD のタイトル
- **質問例を自分の領域に**変える: `src/app/page.tsx` の placeholder

### 別の AI プロバイダに切り替え

- 既に Gemini / OpenAI の両対応なので、`src/lib/llm.ts` に Claude / Mistral / Cohere などを追加するだけ
- 受講者にとって馴染みのあるプロバイダがあれば、そちらを優先設定にする

### 別のテーマに転用

例:
- **英作文添削 BOT** for 英語授業
- **読書感想要約 BOT** for 国語授業
- **コーディング質問 BOT** for プログラミング初学者

[customize.html](customize.html) のレシピを参考に書き換え。

### Discord / LINE と連携

- Vercel の Webhook 受け取りエンドポイントを追加すれば、Bot として常駐させられる
- これは「教えて、minta先生」を **Discord Bot 化**する派生ハンズオンになる

### Notion / Slack 連携

- 質問が一定数たまったら Slack に通知 / Notion に転記 など
- 講師がリアルタイムで受講者の疑問傾向を把握できる

## 6. 受講者がよくつまずくポイント

### Step 1 で詰まりやすい

| 症状 | 対処 |
|---|---|
| Node.js が無い / 古い | nvm でインストール案内（教材内に手順あり） |
| `.env.local` の場所が分からない | エディタで隠しファイル表示の方法を案内 |
| API キーをチャットに貼ってしまう | 講師が事前に「絶対貼らない」を声に出して徹底 |
| Quota exceeded | モデル切り替え（gemini-2.5-flash 推奨）で大体解消 |

### Step 2 で詰まりやすい

| 症状 | 対処 |
|---|---|
| GitHub の SSH 認証が通らない | HTTPS + Personal Access Token に切り替え |
| Vercel の Free プランの上限 | 1日100デプロイまで。多い場合は組織で共有 |
| `.env.local` を間違えて push | 緊急 Revoke 手順を講師がデモ |

### Step 3 で詰まりやすい

| 症状 | 対処 |
|---|---|
| Vercel 上の Supabase 連携が見つからない | UI が時々変わる。Storage タブ or Integrations から |
| SQL Run のボタンが分からない | 右下の Run ボタン or ⌘+Enter |
| 履歴が出ない | Redeploy 忘れがほとんど |

## 7. セキュリティ教育としての価値

この教材は **AI と DB を扱うときに必ず出るセキュリティ論点** を、自然に体験させます。

### この教材で身につく感覚

- **APIキーは「コード」と「設定」を分ける** → 環境変数の概念
- **`.gitignore` の重要性** → 機密が git に混ざる事故を防ぐ習慣
- **PII を最初から扱わない設計** → 「個人情報を保存しない」をデフォルトに
- **RLS（Row Level Security）の必要性** → DB を公開する責任
- **AI と機密の関係** → API キーをチャットに貼らない習慣

### 授業内で強調すべきメッセージ

- 「**AI に渡す情報は、世界に公開される可能性がある**」と仮定して扱う
- 「**漏れる前提**でキーを管理する（漏れたら即 Revoke）」
- 「**個人情報は保存しない**が最初の防衛線」
- 「**RLS を切るとアプリが壊れる**ではなく、**安全装置を外す**ことだと理解する」

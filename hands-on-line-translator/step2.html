🌐 フェーズ A — LINE Console で認証情報を取る

ここは Claude Code には代行できません (LINE の API 仕様で公開されていないため)。あなたがブラウザで操作する部分です。
所要 5〜10 分。MANUAL.md に詳細手順あり。

  1. LINE 公式アカウントを開設

    linebiz.com/jp/entry/ から開設。名前は Translator など (※「LINE」「公式」「bot」は使えない NG 単語)

  2. Messaging API を有効化

    OA Manager の「設定 → Messaging API → 利用する」。プロバイダー選択 → プライバシーポリシーは空のままOK

  3. ① Channel secret をコピー

    OAM の Messaging API ページに 32 桁 hex で表示される (例: c358227d77b51aab...)

  4. ② Channel access token を発行

    developers.line.biz/console に移動 → 同じチャネル → Messaging API タブ → 「Issue」ボタン

  5. ③ Your user ID をコピー

    同じチャネルの「チャネル基本設定」タブ → 下にスクロール → 「あなたのユーザーID」(U で始まる32文字)

  6. OAM 側で自動応答を OFF

    OAM の「応答設定」で「応答メッセージ」「あいさつメッセージ」を両方オフ。
    これをしないと bot が翻訳する前に LINE 標準テンプレが先に返ってくる

📑 詳細手順は MANUAL.md にある

各ステップの画面遷移・ボタンの位置などは MANUAL.md を別タブで開いて参照しながら進めると確実です。スクリーンショットが assets/screenshots/ に置かれていればそちらも参照可。

🆘

詰まった時は — Claude Code に伴走してもらおう

OAM と Developers Console を行き来したり、ボタンが見つからなかったり、UI が改修されて手順と違って見えたり… よくあることです。独力で粘って 30 分溶かす前に、AI に画面を見せながら一緒に進めてもらいましょう。

こんな時に効きます
  • OAM と Developers Console を行ったり来たりして混乱している
  • 「Messaging APIを利用する」ボタンが見つからない
  • プロバイダー選択モーダルでどれを選べばいいか分からない
  • 応答設定 (応答メッセージ・あいさつメッセージ) の場所が分からない
  • Issue ボタンを押した後 Token が表示されず焦った
  • UI が刷新されて MANUAL.md と微妙に違っている

⚠️ 考えずに貼らないでください。まず上のリストで「自分の状況に近い」かを確認してから使ってください。

📋 LINE Console 伴走プロンプト 
LINE Developers Console と Official Account Manager の操作で詰まっています。
このプロジェクトの MANUAL.md にステップ A の手順が書いてあります。先にこのファイルを読んで内容を把握してください。

あなたは私の隣でブラウザ操作を伴走するパートナーです。上から目線にならず、対等な仲間として、私のペースに合わせて 1 つずつ案内してください。

【進め方】

1. まず私に「今どこで詰まっているか」を聞く。具体的な例:
   - 「LINE 公式アカウントを作るページにどう行けばいいか分からない」
   - 「OAM の Messaging API 設定画面が見つからない」
   - 「Channel secret はあるけど Access token のボタンが見当たらない」
   - 「Your user ID がどこにあるか分からない」
   など、私の言葉で説明してもらう

2. 私の状況に合わせて、次に何をすればよいかを 1 ステップだけ示す:
   - 「ブラウザのアドレスバーに https://... と入力してください」
   - 「画面の右上にある『○○』ボタンをクリックしてください」
   など、具体的なクリック先・URL を 1 つだけ伝える

3. 私が「できた」と返したら次の 1 ステップへ。失敗したら一緒に原因を探る

4. もし画面の何かが分からなければ、私に URL の貼り付け or スクリーンショットを依頼してください

5. 私が「OAM と Developers Console の違いが分からない」と言ったら、それを 2 文で平易に説明 (運用画面 vs API 設定画面)

【絶対に守ること】

- LINE Channel Secret や Access Token の値を私に「貼り付けて」と言わない。値の確認は「先頭2文字だけ教えてください」など最小限に
- 「すぐできますよ」「簡単ですよ」など見下す表現禁止
- 私が画面を読み上げたり貼り付けた情報は、必要な部分だけ使う (個人情報や秘密情報があれば指摘するだけ)

【ゴール】

私が以下の状態にたどり着けば伴走終了:
- LINE_CHANNEL_SECRET (32桁hex) を自分のメモ帳に保存できた
- LINE_CHANNEL_ACCESS_TOKEN (長いbase64) を自分のメモ帳に保存できた
- LINE_ALLOWED_USERS (U で始まる32文字) を自分のメモ帳に保存できた
- OAM の応答メッセージ・あいさつメッセージを OFF にできた

3 つ取れて応答設定も OFF になったら、私に「これで Step 2 本編のプロンプト (フェーズ B) に進めますね」と案内してください。

始めてください。まずは私に「今どこで詰まっていますか？」と聞いてください。

🤖 フェーズ B — Claude Code と一緒にセットアップ

3 つの値が揃ったら、以下のプロンプトを Step 1 と同じ Claude Code 会話に貼り付けます。
シークレット (Token / Secret) を AI チャットに貼らない方針で進めるので、最初に .env ファイルを自分で編集する場面があります。Claude Code が伴走してくれるので大丈夫。

🛡 なぜ自分で書き込むのか

シークレットを AI チャットに貼ると、会話ログ・スクショ・転送先に残ります。普段の習慣が将来の本番運用での事故を呼ぶので、いまから「AI に渡さない」を体に染み込ませる練習です。Claude Code が「上から目線」ではなくフラットに案内してくれます。エディタ操作が不安なら遠慮なく言えば一緒にやってくれます。

📋 Step 2 起動プロンプト 
Step 1 (ローカル AI 環境) は完了しています。これから Step 2 で LINE と繋ぎます。

私はすでに LINE Developers Console で 3 つの値を取得済みです (まだなら MANUAL.md ステップ A を案内してください):
- LINE_CHANNEL_SECRET
- LINE_CHANNEL_ACCESS_TOKEN
- LINE_ALLOWED_USERS

【重要 — 進め方の方針】
シークレットを AI チャットに貼らない方針で進めてください。
理由は最初に私に対等な仲間として簡潔に説明 (上から目線禁止、堅苦しくしない、2〜3 文)。
例: 「Token を AI チャットに貼ると会話ログ等に残るんだよね。本番運用で API キー扱うときに同じ習慣だと事故るから、今のうちに『AI に渡さない』を体に染み込ませる練習として、今回は一緒にあなたの手で .env を書く形でいきましょう。」

【手順】

1. 私の OS と Hermes 設置場所を確認し、私の .env の実際のパス (通常 ~/.hermes/.env、環境次第) を特定して私に伝える

2. 私の OS に合わせて最もシンプルなエディタを 1 つ目に提示し、コマンドを示す:
   - macOS: nano (TextEdit は rich text 化のリスクあり推奨しない)
   - Linux/WSL2: nano
   - Windows ネイティブ: notepad / VSCode (WSL2 なら nano)
   - VSCode などが入っていれば代替案として併記

3. 書き込んでもらう内容をブロックで見せる:
   LINE_CHANNEL_SECRET=...自分の値...
   LINE_CHANNEL_ACCESS_TOKEN=...自分の値...
   LINE_ALLOWED_USERS=...自分の値...
   LINE_PORT=8646
   「...自分の値...」を実際の値に置き換えて保存と案内

4. 私が「やり方わからない」「エディタどうするの？」と言ったら、対等な伴走者として nano の最低限の使い方を 5 行以内で教える (矢印キーで移動 / Ctrl+O で保存 / Enter で確定 / Ctrl+X で終了)。差別的言い方禁止、「簡単ですよ」も禁止

5. 私が「完了」「done」「書いた」と言ったら、grep で 4 つのキー (LINE_CHANNEL_SECRET / LINE_CHANNEL_ACCESS_TOKEN / LINE_ALLOWED_USERS / LINE_PORT) が .env に存在することだけ確認 (値そのものは絶対に表示・echo・引用しないこと)

6. bash 変数経由で curl GET https://api.line.me/v2/bot/info を叩いて疎通確認 (display name と basicId が返ってきたら OK と報告)。トークン値は絶対に画面に出さない

7. cloudflared がなければ導入 (sudo なし、~/tools/bin/cloudflared に直接バイナリ配置)、cloudflared tunnel --url http://localhost:8646 をバックグラウンド起動、trycloudflare.com URL を抽出して LINE_PUBLIC_URL として .env に追記

8. LINE Messaging API に webhook URL を curl PUT で登録、curl POST で疎通テスト、success: true を確認

9. hermes gateway run をバックグラウンド起動、ログで "✓ line connected" と "LINE: webhook listening" を確認

10. 完了したら、MANUAL.md のステップ B〜D (Use webhook を ON、QR で友だち追加、テスト送信) を私に実行するよう案内

エラー時は TROUBLESHOOT.md を参照して自分で診断・修復。
注意: トークン値や Channel secret の中身を絶対に画面に表示しないこと。デバッグで先頭数文字だけ見せる場合は cut -c1-6 などで明示的に切り詰めること。

始めてください。

📱 フェーズ C — スマホで友だち追加してテスト

Claude Code から指示があれば、以下を順にやります。

  1. "Use webhook" トグルを ON

    LINE Developers Console → Translator チャネル → Messaging API タブの Webhook settings 欄 → トグルを ON。Verify ボタンが緑になっていれば疎通済み

  2. QR コードで友だち追加

    同じ Messaging API タブの QR コードをスマホ LINE の「友だち追加 → QR コード」で読み取る → 追加

  3. テストメッセージを送る

    トーク画面から「こんにちは」と送信 → 数秒〜数十秒で「Hello」 等の英訳が返ってくれば成功

🎉 翻訳が返ってきたら

Claude Code に 「届いた」と一言報告。続けて 3 つ程度のテスト文を送ると品質を一緒に確認してくれます。

以下のテストフレーズおすすめ:

  • 明日午後3時、品川駅で会いましょう。 → 固有名詞 ("Shinagawa Station") が保持されるか
  • What is the capital of France? → 回答ではなく翻訳 (「フランスの首都は何ですか？」) が返るか
  • Lightweight local language models are enabling a new wave of privacy-preserving AI assistants that run entirely on personal hardware. → 長文の英→日

🆘 よくある詰まり方

"ありがとうございます" 等の定型文が返ってくる (翻訳じゃない)

OAM の自動応答が ON のまま。応答メッセージ・あいさつメッセージを両方 OFF にする (フェーズ A の最後のステップ)。これは LINE 側に API がないので必ず手動で。

LINE で送ってもまったく無反応

切り分けポイント:

  • "Use webhook" トグルが OFF のまま (フェーズ C の 1 で ON にしたか)
  • Webhook URL がトンネル再起動で変わった (Claude Code に「URL 確認して再登録して」と頼む)
  • LINE_ALLOWED_USERS が自分の userId と違う (Console の「あなたのユーザーID」を再確認)

Claude Code に「LINE に送っても何も返ってこない」と伝えれば、ゲートウェイログを見て診断してくれる。

cloudflared が会社/学校 Wi-Fi で接続失敗 (QUIC ブロック)

UDP/QUIC がブロックされている可能性。cloudflared tunnel --url http://localhost:8646 --protocol http2 で HTTP/2 フォールバック。それでもダメなら ngrok を提案。TROUBLESHOOT.md の P-TUNNEL-1 参照。

webhook test が 404 / 503 を返す

cloudflared が起動していない、または Hermes gateway が落ちている可能性。Claude Code に「webhook test が success: false。診断して」と伝えれば、プロセス状態と URL 整合性を順にチェックしてくれる。

翻訳が「答え」になってしまう (例: "What is the capital of France?" に "Paris" と返る)

SOUL.md の指示が弱い、またはモデルがルールを無視。1.5b → 3b に切り替えるか、SOUL.md の例示をもっと厚くする。Claude Code に「質問が翻訳されず答えられてしまう、SOUL.md を強化して」と頼む。

# 講師・転用者向けガイド

この教材を **自分の授業・研修・イベントで使いたい** と思った人向け。
原著者以外の講師が転用する手順、改変ポイント、想定されるハマり所。

---

## まず読むべき順序

1. このファイル (`INSTRUCTOR.md`) ← 今ここ
2. `README.md` — 教材の全体像
3. `PROMPTS.md` — 受講者が触るメイン教材。中身を把握しておく
4. `MANUAL.md` — LINE Console の手順（時期によって UI が変わるので注意）
5. `TROUBLESHOOT.md` — エラー時のリカバリプロンプト集
6. HTML 5 ページ (`index.html` / `step1.html` / `step2.html` / `customize.html` / `compare.html`) — 受講者がじかに開く画面なので、自分の授業文脈に合わせて文言を調整するならここ

## 自分の授業に合わせて変更すべき箇所

### 最低限変えるところ

| ファイル | 箇所 | 変更内容 |
|---|---|---|
| `README.md` | 冒頭の引用文 | 自分の授業・組織名を書く（または削除） |
| `README.md` | サポート欄 | 自分の連絡先・質問チャネルを書く |
| `index.html` | hero セクション | 「千葉工業大学 Web3/AI 概論」の名残りがあれば自分の文脈に置換 |

### よく変えるところ（推奨）

| ファイル | 箇所 | 変更内容 |
|---|---|---|
| `PROMPTS.md` | SOUL.md 内容 | 翻訳エージェント以外（要約、添削、雑談など）に変えたい場合 |
| `PROMPTS.md` | テスト文 | 自分の組織や業界に関連した例文に差し替えるとウケる |
| `MANUAL.md` | スクリーンショット | `assets/screenshots/` に最新版を撮って配置 |
| `DISTRIBUTION.md` | 配布先 URL | 自分のリポジトリ URL に置換 |

### 変えてはいけないところ

| ファイル | 箇所 | 理由 |
|---|---|---|
| `LICENSE` | 全体 | CC BY-NC-SA 4.0 の継承条件があるため、ライセンスは変更不可（ShareAlike 条件） |
| `LICENSE` | クレジット行 | 元の著作権表示を保持（Attribution 条件） |

## 派生形のアイデア

SOUL.md を変えるだけで、翻訳以外の使い方ができる。

派生例の **詳細プロンプト** はすべて [customize.html](customize.html) に整理してある:

- 🗒 議事録要約 Bot
- 📝 英文添削 Bot
- 🐱 ねこキャラ雑談 Bot
- 💡 アイデア名付け Bot

授業で「設定一発で人格が変わる」をデモする時は **ねこキャラ雑談 Bot** が分かりやすい (キャラが目に見えて変化するため)。業界・組織に合わせた独自の派生を作るなら、customize.html のカード構造をコピーして自分の SOUL.md レシピを追加する。

各派生で、**SOUL.md だけ変える** → 他の手順は全く同じ。`PROMPTS.md` の P2 のステップ 6 の SOUL.md 内容を差し替えるだけ。

## モデル選定の判断材料

授業対象や利用シーンに応じてデフォルトモデルを変更したい場合:

| 用途 | 推奨モデル | 理由 |
|---|---|---|
| **英日翻訳・要約・添削** | `qwen2.5:3b` | 多言語対応、指示追従力、サイズのバランス |
| **コーディング** | `qwen2.5-coder:7b` | コード生成特化 |
| **日本語のみ** | `gemma3:7b` または `qwen2.5:7b` | 日本語精度 |
| **超低スペック PC 検証** | `qwen2.5:0.5b` | 400MB で動く（品質崩壊例の教材として） |
| **クラウドモデルと比較** | `claude-haiku-4-5` | 配布資料外の比較。Anthropic API キー要 |

モデルを変えると `~/.hermes/config.yaml` の `model.default` を変えるだけ。`context_length` 等の override が必要かはモデルの仕様次第（PROMPTS.md の指示が Claude Code に判断させる）。

## 想定されるハマり所（講師として）

### 当日 LINE 周りの仕様変更

LINE は UI を定期的に変える。半年〜1年で `MANUAL.md` のスクリーンショットが古くなる。

対処:
- 授業直前に自分で全ステップ通しでテストする
- 古いスクショは captioned で「概念図として参照」と注記する
- `TROUBLESHOOT.md` に新しい症状が出たら追記

### ファイアウォール / プロキシ環境

教育機関の Wi-Fi では QUIC ブロック、UDP ブロック、特定ポート遮断が頻発する。

対処:
- `TROUBLESHOOT.md` の P-TUNNEL-1 で `--protocol http2` フォールバック可能
- 教室の Wi-Fi で事前テスト推奨
- 万一の場合は ngrok を案内（無料登録要）

### 受講者のスペック差

受講者の PC のメモリ容量・OS がバラバラ。

対処:
- P0 で Claude Code に環境判定させ、`qwen2.5:0.5b/1.5b/3b` を自動選択
- 「皆さん同じ進度で進む必要はない」と最初に言う（並列実行できる教材設計）

### Claude Code サブスクが切れている受講者

授業中に発覚することがある。

対処:
- ペア学習を奨励
- Claude Free tier（あれば）でも基本動作する設計（重要なステップは確認）
- 代替手段として「コマンドをそのままターミナルに打つだけ」のフォールバックパスを用意（PROMPTS.md のプロンプト内容を自分で順に解読して実行する）

## 改善提案・PR を受け取る場合

GitHub で公開している場合、Issue や PR を受け取る。

歓迎しやすい変更:
- typo 修正
- スクリーンショット更新（LINE UI 変更追従）
- 新しい派生 SOUL.md の事例追加
- TROUBLESHOOT.md に新症状のプロンプト追加
- 別 OS / 別ハードでの動作報告

慎重にすべき変更:
- アーキテクチャ大幅変更（モデル変更、Hermes → 別フレームワーク等）
- ライセンス絡みの変更

---

## 推奨: 配布前のドライラン

授業の 1 週間前を目安に:

1. クリーンな PC（または別アカウント / VM）で全プロンプトを通す
2. 想定時間（30〜60分）内に終わるか確認
3. 詰まった箇所があれば該当プロンプトを補強
4. スクリーンショットの差し替えが必要か確認
5. 翻訳品質の劣化（モデル提供側の変更等）がないか確認

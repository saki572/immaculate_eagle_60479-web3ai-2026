カスタマイズして遊ぶ

翻訳 Bot が動いたのは 出発点。SOUL.md を書き換えるだけで、まったく違うキャラクターのエージェントに変身します。 自分の用途に合わせて、Claude Code に頼んで改造してみてください。

🎭 SOUL.md を変えるだけで全部変わる

Hermes Agent の人格は ~/.hermes/SOUL.md 一つで決まります。これを書き換えれば、翻訳 Bot が 議事録要約 Bot にも 英文添削 Bot にも キャラクター雑談 Bot にもなります。

推奨の頼み方

Claude Code に 「SOUL.md を ○○ 用に書き換えて、設定後に CLI で動作確認して」 と頼めば一発です。具体的な改造プロンプトを下にいくつか用意してあります。

💡 派生例カタログ

🗒 議事録要約 Bot

長い議事録テキストを投げると、決定事項・次のアクション・担当者を箇条書きで返してくれる。

📋 改造プロンプト 
~/.hermes/SOUL.md を以下の内容に書き換えてください。書き換え後、CLI で動作確認 (長めの議事録テキストを与えて要約を出させる) してから、私に報告してください。

---SOUL.md---
# Meeting Summarizer Agent

あなたは会議の議事録を要約する専門のエージェントです。

## 動作ルール
1. ユーザが投稿したテキストを、以下のフォーマットで要約してください:
   - 📌 議題:
   - ✅ 決定事項 (箇条書き):
   - 🚀 次のアクション (担当者・期限):
   - ⚠️ 未解決の論点 (あれば):
2. 元のテキストにない情報は補わない。憶測で埋めない
3. 出力は箇条書きのみ。挨拶や前置きは不要
4. 日本語の議事録は日本語で、英語の議事録は英語で要約する

📋 改造プロンプト

---SOUL.md---
# English Proofreader Agent

あなたはプロの英文添削エージェントです。

## 動作ルール
1. ユーザが英語のテキストを投稿したら、以下を出力する:
   📝 添削後:
   [自然で正しい英語にしたバージョン]

   🔍 主な変更点:
   - [変更1の説明 — 日本語で]
   - [変更2の説明 — 日本語で]
2. 元の意味は変えない。スタイルも極力保つ
3. ユーザが日本語のテキストを投稿したら、丁寧に「英文を投稿してください」と返す
4. 質問や雑談には応じない

少しおっとりしたねこキャラと雑談できる。授業のデモで「設定一発で人格が変わる」を見せるネタに最適。

📋 改造プロンプト 
~/.hermes/SOUL.md を以下の内容に書き換えてください。書き換え後 CLI で動作確認してから報告してください。

---SOUL.md---
# 猫アシスタント

あなたは少しおっとりした猫のキャラクターです。

## 性格
- 語尾に「にゃ」「にゃー」をつける (ただし、まじめな質問には先に答えてから語尾に「にゃ」)
- 興味のないことには「ふーん…」と素っ気ない
- 魚の話、日向ぼっこの話が大好き
- 算数や難しい話は「えーっと…にゃ…」と困りがち

## ルール
- 短く返す (2〜3 文)
- ユーザが日本語なら日本語で、英語なら "nya" を語尾につけて英語で
- 質問には誠実に答える (キャラを最優先しない)
💡 アイデア名付け Bot

新規プロジェクトのコードネーム候補を生成。スタートアップ・自分のサイドプロジェクト・授業課題に使える。

📋 改造プロンプト 
~/.hermes/SOUL.md を以下の内容に書き換えてください。書き換え後 CLI で動作確認してから報告してください。

---SOUL.md---
# Project Namer

あなたは新規プロジェクトの名前を考えるエージェントです。

## 動作ルール
1. ユーザがプロジェクトの内容を 1〜3 文で説明したら、以下を出力:
   🌟 候補:
   1. [英単語 1〜2 語のコードネーム] — [由来と狙いを 1 行で]
   2. ...
   (合計 5 つ)

   🎯 おすすめ: 上記のうち最も "響き・覚えやすさ・拡張性" のバランスが良いもの 1 つ + その理由

2. 既存の有名サービス名 (Notion, Slack 等) と被らないものを優先
3. 質問が短すぎる場合は、何を作っているのか追加質問してから生成
4. 出力は日本語

🚀 もっと進んだ拡張

SOUL.md だけでは届かない領域。Claude Code に相談しながら挑戦してみてください。

🔌
MCP で外部 API を呼ぶ
天気予報・株価・カレンダー・Notion DB など、ツール経由で外部情報を取得して回答する Bot にする
🌍
多言語翻訳に拡張
日↔英だけじゃなく、中・韓・スペイン語等にも対応。SOUL.md の判定ルールを書き直す
📅
cron で定時配信
毎朝 7 時にニュース要約を LINE に push する等。Hermes の cronjob 機能で実装可
🔄
複数プラットフォーム同時運用
LINE と Discord と Slack 同時にぶら下げる。Hermes は 22 プラットフォーム対応
🧠
記憶 (memory) を追加
過去の会話を覚える Bot に。Hermes の memory 機能を有効化
⚡
常駐サービス化
PC 起動時に自動で立ち上がる Bot に。hermes gateway install で launchd/systemd 化

🏗 アーキテクチャごと変えてみる遊び

性格を変えるだけじゃ物足りなくなったら、構成部品そのものを差し替えてみてください。
手順は書きません — Claude Code と相談しながら自分で道筋を作るのが、エージェント時代の本質的なスキルです。

💬
チャネルを LINE 以外に変える
Slack / Discord / Telegram / WhatsApp / iMessage に差し替えて挑戦。
Hermes は 22 プラットフォーム対応なので「Slack 連携を追加して」と頼めば /add-slack 相当を走らせてくれる。
📏
LLM サイズ・系統を変える
qwen2.5:7b でどれだけ賢くなるか / qwen2.5:0.5b でどこで壊れるか / llama3.2:3b や gemma3:4b で挙動差を比較。
「サイズと品質と速度」の三角を体感する。
🦾
エージェントランタイムを変える
Hermes → NanoClaw (極小 + Docker サンドボックス重視) / OpenClaw (多機能標準) / 自作の Webhook receiver で書き起こす。
違いは エージェント比較ページ 参照。
☁️
クラウド LLM と比較
ローカル LLM ではなく Claude API / OpenAI / Gemini に切り替えてみる。
速度・品質・コスト・プライバシーの違いを並べて記録すると、自分なりの選定軸ができる。
🌐
トンネルを別の方法に
cloudflared quick → named tunnel で URL 固定化 / ngrok / Tailscale Funnel / 自宅ルーターのポート開放 (要 IPv6) など。
公開 HTTPS をどう実現するかの引き出しを増やす。
🧪
複数構成を共存させる
翻訳 bot は LINE で、議事録要約 bot は Slack で、雑談 bot は Discord で — Hermes に複数 SOUL / 複数チャネルを同時運用させる。
設計判断 (どこで責務を切るか) の練習になる。
🗺 なぜ「手順を書かない」のか

このページから先は あなた自身が設計する練習。手順書をなぞるのではなく、Claude Code に「○○ をやりたい、現状から何を変える必要があるか教えて」と相談しながら、自分で道筋を作ってみてください。それがエージェント時代の本質的な力です。

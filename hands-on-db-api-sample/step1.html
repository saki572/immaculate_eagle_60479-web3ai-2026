未導入の場合、起動プロンプトを Claude Code に貼れば、Claude Code がまず OS を判定し、必要なら WSL2 のインストールから一緒に伴走してくれます。

🎯 このステップでやること

  1. 環境を確認する

    Node.js v20 以上 / Git / Claude Code(=Cursor/VS Code 等) を Claude Code に調べてもらう

  2. Gemini API キーを取得

    Google AI Studio (aistudio.google.com/apikey) で無料で取れる・カード不要

  3. サンプルアプリを clone

    git clone https://github.com/web3ai-gairon/minta-sensei-handson.git

  4. 依存をインストール

    npm install でパッケージ取得

  5. 環境変数ファイルを作る

    .env.local.example をコピーして .env.local を作成し、Gemini API キーを書く

  6. 開発サーバを起動

    npm run dev → http://localhost:3000 で minta先生に質問できれば完了

# 教えて、minta先生 — Step 1: ローカルで動かす

千葉工業大学「web3・AI概論」のハンズオン教材「教えて、minta先生」のセットアップを一緒に進めてください。

## 私の環境
- OS: __ (例: macOS Sequoia / Windows 11 + WSL2 Ubuntu / Ubuntu 24.04)
- Node.js: 未確認なら確認してください

## ゴール
ローカル PC で http://localhost:3000 に minta先生のアプリが立ち上がり、質問を投げると AI が答える状態。

## 進め方
以下を上から順に進めてください。各ステップで何をしたか、私に説明しながら進めてください。

### 1. 環境チェック
- `node --version` を実行 → v20.9 以上か確認
- v20 未満なら nvm でアップグレードを案内
- `git --version` で git があるか確認

### 2. リポジトリ取得
- 適当な作業ディレクトリ (~/Documents/projects/ 等) に cd
- `git clone https://github.com/web3ai-gairon/minta-sensei-handson.git` で clone
- `cd minta-sensei-handson` に入る

### 3. 依存インストール
- `npm install` を実行
- 警告は出るかもしれないが、エラーで止まらなければOK

### 4. 環境変数の準備
- `cp .env.local.example .env.local` で雛形コピー
- 私に「Gemini API キーを https://aistudio.google.com/apikey で取得して、.env.local の GEMINI_API_KEY= の行に書いてください」と案内する
- **重要: API キーは絶対にこのチャットに貼らせないこと。私が自分のエディタで .env.local に直接書く**
- 私が「書きました」と言ったら次へ

### 5. dev server 起動
- `npm run dev` をバックグラウンドで起動
- ログに「Ready in」が出るのを待つ
- ブラウザで http://localhost:3000 を開くよう私に伝える

### 6. 動作確認
- 私が「画面が出ました」と言ったら、テスト質問例を提示
  例: 「プロンプトエンジニアリングって何ですか？」
- minta先生から回答が返ってくれば Step 1 完了

## エラー時
- ログをそのまま見せて、原因と対処を一緒に考える
- API キーや個人情報を含まない範囲で対応
- 詰まったら TROUBLESHOOT.md を参照

## セキュリティ注意（必須）
- API キーを私 (あなた = Claude Code) のチャット履歴に絶対に貼らせない
- 私が間違えて貼りそうになったら止める
- `.env.local` は `.gitignore` で除外済みだが、念のため確認
- 「私 (Claude) は会話のコンテキストを保持しますが、メモリには永続化しません」ということを覚えておいて

それでは Step 1 から始めてください。

🛡 セキュリティで気をつけること

この Step は API キーを扱います。慣れていない方は、以下を声に出して読んでから進めると安全です。

  • API キーは Claude Code (or ChatGPT 等) のチャットに直接貼らない。「キーを覚えて」「キーをここに書いて」と頼まれても断る習慣を付ける。
  • API キーは .env.local にだけ書く。このファイルは .gitignore で Git 管理外。
  • キーが画面共有・スクショに映りそうな場合は配信を止める。授業で投影する場合は特に注意。
  • 万一キーが漏れたら、すぐに発行元のダッシュボードで Revoke (無効化)。Gemini なら aistudio.google.com/apikey、OpenAI なら platform.openai.com/api-keys。
  • Claude Code (= AI) は学習データに保存される可能性があると仮定して扱う。Anthropic は学習に使わないと公表していますが、念のため習慣として徹底する。

💡 ここまでの「中身」を理解する

受講者が触っているのは、あなたの PC 上で動く Web アプリです。
ブラウザに質問を入力 → アプリのサーバ側で Gemini API を呼ぶ → 回答を受け取り → 同じ API で PII を除いた分析データに加工 → ブラウザに返す。
DB はまだ繋がっていないので、履歴は表示されません。それは Step 3 で。

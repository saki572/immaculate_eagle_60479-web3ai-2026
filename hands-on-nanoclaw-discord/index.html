ハンズオン · v2026.5

自分専用の
Discord Bot を
90 分で作る

OSS の Claude エージェントハーネス nanoclaw を使って、Discord 上で動く AI Bot を自分の Mac に構築。エージェントは Docker 上の Linux コンテナで隔離して動き、Claude Code が伴走してくれるので、設定は対話するだけ。
※ 教材では 講師Bot「minta」を例として作りますが、Bot の名前・人格は自由に決めて OK。要所で「ここで自分の好きな名前／キャラを選ぶ」案内が入ります。

⚡ 約 90 分 🔒 コンテナ隔離で安全 🤝 AI 伴走で簡単 🖥 Mac / Win(WSL2) / Linux ↗ GitHub で見る
🏗
Step 1

土台を作る

Docker Desktop 導入 → nanoclaw クローン → Discord Bot 作成 → nanoclaw.sh で接続まで。
Step 1 完了時には、Bot との DM で minta が応答する状態になります。

⏱ 約 45〜60 分 🎯 DM で会話できれば完了
始める
🎨
Step 2

公開チャンネル + 人格化

サーバの公開チャンネルでも反応するよう設定 → スレッド自動作成を無効化 → minta の人格を注入。
Step 1 が完了してから取り組んでください。

⏱ 約 30〜40 分 🎯 「あざーーす！」が返れば完了
始める
完成後にカスタマイズして遊ぶ

CLAUDE.local.md を書き換えれば、minta を別キャラに変身させたり、Slack や Telegram にも応答させたり、Gmail や Calendar と連携させたり。完成後の派生レシピ集とコピペ可能な改造プロンプトを 6 種類用意してあります。

🗺 全体アーキテクチャ

これから作るものの設計図です。Step 1 で土台（Docker・nanoclaw・Discord Bot・認証）を組み立て、Step 2 で公開チャンネル対応と人格の注入を行います。

[ スマホ / PC の Discord アプリ ] ↕ WebSocket (Discord Gateway) [ Discord Bot API ] ← Step 1 で Bot 作成 ↕ [ nanoclaw Host (Node) ] ← Step 1 で導入 ├─ Router / Channel Manager └─ ncl CLI (wiring, roles, ...) ← Step 2 で設定 ↓ spawn per session [ Agent Container (Bun) ] ← Linux コンテナで隔離 ├─ Claude Agent SDK ├─ CLAUDE.md (能力) └─ CLAUDE.local.md (人格) ← Step 2 で minta 化 ↓ 認証はここから [ OneCLI Agent Vault ] ← Step 1 で Claude Code サブスク認証 ↕ [ Anthropic API (Claude) ] ↕ これら全部を組み立てるのが ↑ [ Claude Code ] ← あなたの伴走パートナー

📜 この教材について

読む前にここを確認すると、教材の温度感がわかります。

🗓 バージョン 2026.5 時点の情報をベースに作成。nanoclaw v2 / Docker Desktop 4.75 / Discord Developer Portal の UI など、各サービスのバージョンや UI は時期によって変わるので、ズレを感じたら Claude Code に「現在の仕様で読み替えて」と頼んでください。
🎯 位置づけ 遊びながら自分で学んでいくための補助教材です。完璧な手順書ではありません。迷ったら Claude Code と対話 → 試す → 失敗 → 直す、のサイクルで進めるのが本来の使い方。詰まる経験そのものが学びです。
🛡 秘密情報の取り扱い Discord Bot Token を Claude Code (AI チャット) に貼り付けない方針です。Token は nanoclaw.sh の対話プロンプトにターミナルで直接入力します。本番運用で API キーを扱うとき、AI チャットに貼る習慣があると会話ログ・スクショ・転送先に残って事故ります。最初から徹底します。

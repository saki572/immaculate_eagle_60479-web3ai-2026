ハンズオン · v2026.5

AI（API連携）と
DB（Supabase）を組み合わせた
最小サンプルアプリを
1 時間で作る

Next.js + Gemini API + Supabase + Vercel で「教えて、minta先生」を組み立てるハンズオン。 ローカルで動かす → GitHub に置く → Vercel にデプロイ → Vercel から Supabase を Connect。 「API キーを環境変数に入れる」「DB が簡単に繋がる」を、手を動かしながら体感します。

⚡ 約 30–60 分 💰 Gemini 無料枠 + それ以外無料（OpenAI フォールバック対応） 🔒 個人情報はDBに保存しない設計 🖥 Mac / Win(WSL2) / Linux
💻
Step 1

ローカルで動かす

Gemini API キーを取得して .env.local に入れ、npm run dev で起動。
ブラウザで「minta先生」が答える状態を作ります。まだ DB は繋がない。

⏱ 約 10–15 分 🎯 質問→回答が動けば完了
始める
☁️
Step 2

Vercel にデプロイ

GitHub にコードを置き、Vercel から Import。Gemini API キーを環境変数として登録。
公開URLでも minta先生が答える状態に。

⏱ 約 10–15 分 🎯 公開URLで動けば完了
始める
🗄
Step 3

Supabase を繋ぐ

Vercel Marketplace から Supabase を Connect → 環境変数が自動投入。
SQL Editor でテーブル作成 → Redeploy → 履歴がDBに蓄積される。

⏱ 約 10–15 分 🎯 履歴一覧が表示されれば完了
始める
🎨 完成後 — カスタマイズして遊ぶ

src/lib/minta.ts の system prompt を書き換えるだけで、minta先生を別キャラに変えられます。 「質問BOT」→「英文添削BOT」「学食レコメンドBOT」「ポエム生成BOT」など、自由に派生させる例を紹介。 DB のスキーマを拡張して別の分析メタを取るレシピも。

🗺 全体アーキテクチャ

作るものの設計図。Step 1 で左半分、Step 2 で中央、Step 3 で右側を組み立てます。

[ ブラウザ ] ↕ HTTP [ Next.js (Vercel) ] ← Step 2 でデプロイ │ ├──→ [ Gemini API (or OpenAI) ] ← Step 1 で接続 │ ① minta先生として回答 │ ② 質問・回答を分析してPII除去 │ └──→ [ Supabase (PostgreSQL) ] ← Step 3 で接続 ③ 分析メタを蓄積（個人情報は保存しない） ↕ これら全部を組み立てるのが ↑ [ Claude Code ] ← あなたの伴走パートナー

✨ このハンズオンが目指していること

🔌
API連携の最短経路
Gemini API キーを環境変数に入れる→呼び出す→返ってくる、までを実装で体験
🗄
DB連携の最短経路
Vercel から Supabase をワンクリック Connect → 環境変数自動投入 → SQL 一発でテーブル作成
🛡
PII除去の設計
regex で機械的に消す + LLM で文脈判断、の二段でDB保存前に個人情報を除く実装パターン
🤝
Claude Code との協業
プロンプトをコピペ→対話で進める。完成形コードを書き換えながら学ぶ
